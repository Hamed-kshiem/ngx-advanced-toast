ngx-advanced-toast

A lightweight, fully-typed Angular toast notification library built on the native <dialog> element. Uses Angular Signals for reactivity, CSS-only animations, and CSS custom properties for theming — no RxJS, no third-party deps.

Installation

npm install ngx-advanced-toast

Peer dependencies: @angular/core ≥ 21, @angular/common ≥ 21.

Setup

Register the provider in your app.config.ts:

// app.config.ts
import { provideToast } from 'ngx-advanced-toast';

export const appConfig: ApplicationConfig = {
  providers: [
    provideToast({
      position: 'top-right',
      duration: 4000,
      maxVisible: 5,
      dismissible: true,
      showProgress: true,
    }),
  ],
};

Then add the container to your root template:

<!-- app.html -->
<ngx-toast-container />
<router-outlet />

Import ToastContainerComponent in your root component's imports array if using standalone components.

Configuration — provideToast(config?)

All options are optional. Unset options fall back to the defaults shown below.

Option Type Default Description
position ToastPosition 'top-right' Default screen position for all toasts.
duration number 4000 Auto-dismiss delay in milliseconds. Set to 0 for persistent toasts.
maxVisible number 5 Maximum toasts rendered simultaneously. Extra toasts are queued.
dismissible boolean true Whether toasts can be manually dismissed by clicking the × button.
showProgress boolean true Show an animated progress bar indicating remaining display time.
animationDuration number 300 Enter/leave animation duration in milliseconds.
gap number 12 Vertical gap between stacked toasts in pixels.

ToastService

Inject ToastService anywhere in your app:

import { ToastService } from 'ngx-advanced-toast';

@Component({...})
export class MyComponent {
  private toast = inject(ToastService);
}

Methods

Method Signature Description
show() show(options: ToastOptions): string Show a fully customised toast. Returns the toast id.
success() success(message: string, options?: Partial<ToastOptions>): string Show a success toast.
error() error(message: string, options?: Partial<ToastOptions>): string Show an error toast.
warning() warning(message: string, options?: Partial<ToastOptions>): string Show a warning toast.
info() info(message: string, options?: Partial<ToastOptions>): string Show an info toast.
loading() loading(message: string, options?: Partial<ToastOptions>): string Show a persistent loading toast (spinner icon, duration: 0). Returns id for later update().
update() update(id: string, options: Partial<ToastOptions>): void Update an existing toast in-place — type, message, duration, etc. Ideal for loading → success/error transitions.
dismiss() dismiss(id: string): void Dismiss a specific toast by id with the leave animation.
dismissAll() dismissAll(): void Dismiss all visible toasts and clear the queue.

Signals (read-only)

Property Type Description
visibleToasts Signal<Toast[]> All toasts currently rendered (up to maxVisible).
toastsByPosition Signal<Map<ToastPosition, Toast[]>> Visible toasts grouped by position — used internally by ToastContainerComponent.
queueLength Signal<number> Number of toasts waiting in the queue behind the visible limit.

ToastOptions

Passed to show(), or as the second argument to success() / error() / etc. Per-toast options override the global config for that toast only.

Option Type Required Description
message string ✓ Main notification text.
type ToastType Visual style. Defaults to 'default'.
title string Optional bold heading above the message.
duration number Override auto-dismiss delay (ms). 0 = persistent.
position ToastPosition Override screen position for this toast.
dismissible boolean Override whether the × dismiss button is shown.
showProgress boolean Override progress bar visibility for this toast.
actions ToastAction[] Interactive action buttons rendered below the message.
id string Custom id. Auto-generated if omitted (toast-{timestamp}-{random}).
data unknown Arbitrary metadata attached to the toast (not displayed).

ToastAction

Define interactive buttons inside a toast via the actions option.

toast.show({
  type: 'warning',
  message: 'Delete this item?',
  duration: 0,
  actions: [
    {
      label: 'Delete',
      style: 'danger',
      callback: (toastId) => performDelete(),
    },
    {
      label: 'Cancel',
      style: 'secondary',
      callback: () => {},
    },
  ],
});
Property Type Required Description
label string ✓ Button text.
callback (toastId: string) => void ✓ Called when the button is clicked. Receives the toast's id.
style ToastActionStyle Visual variant: 'primary', 'secondary', or 'danger'. Defaults to 'secondary'.
closeOnClick boolean Auto-dismiss the toast when the action is clicked. Default: true.

Loading → Update Pattern

Use loading() to show an indeterminate spinner, then update() to transition it to a result state. Useful for async operations like API calls. 

const toast = inject(ToastService);

async function saveData() {
  const id = toast.loading('Saving changes...', { title: 'Please wait' });

  try {
    await api.save(data);
    toast.update(id, {
      type: 'success',
      title: 'Saved!',
      message: 'Your changes have been saved.',
      duration: 4000,
      showProgress: true,
    });
  } catch (err) {
    toast.update(id, {
      type: 'error',
      title: 'Save failed',
      message: 'Could not save changes. Please try again.',
      duration: 0,
      actions: [{ label: 'Retry', style: 'primary', callback: saveData }],
    });
  }
}

ToastLogger

ToastLogger is a thin wrapper around ToastService that simultaneously displays a toast and writes a structured message to the browser console — useful for debugging, error monitoring, or audit logging. 

import { ToastLogger } from 'ngx-advanced-toast';

@Component({...})
export class MyComponent {
  private logger = inject(ToastLogger);

  handleError(err: HttpErrorResponse) {
    this.logger.error('Request failed', {
      title: 'API Error',
      data: { status: err.status, url: err.url },
    });
    // Console output:
    // [TOAST][ERROR] API Error: Request failed
    // { status: 500, url: '/api/users' }
  }
}

ToastLogger exposes the same methods as ToastService:

Method Console method Extra parameter
success() console.log data?: unknown — logged after the message
error() console.error data?: unknown
warning() console.warn data?: unknown
info() console.info data?: unknown

Pass data in the options object: 

logger.error('Upload failed', { data: { fileSize: 104857600, limit: 52428800 } });
