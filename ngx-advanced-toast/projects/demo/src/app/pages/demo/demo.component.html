Toast Types

Loading → Success Pattern

Shows a loading toast then updates it to success after 2.5s — ideal for async operations.

Action Buttons

Toasts can include interactive action buttons (primary, secondary, danger styles).

Persistent & Custom

ToastLogger

ToastLogger wraps the service and also logs to the browser console. Open DevTools → Console to see the output.

Queue & Stacking

Max 5 toasts visible — overflow is queued and promoted as toasts dismiss.

Quick API Reference

// app.config.ts
providers: [
  provideToast({ position: 'top-right', duration: 4000 })
]

// In a component or service
const toast = inject(ToastService);
toast.success('Saved!', { title: 'Done' });
toast.error('Failed', { duration: 0 });
toast.warning('Session expiring');
toast.info('Update available');

// Loading → update pattern
const id = toast.loading('Saving...');
await save();
toast.update(id, { type: 'success', message: 'Saved!', duration: 4000 });

// With actions
toast.show({
  type: 'warning',
  message: 'Delete item?',
  duration: 0,
  actions: [
    { label: 'Delete', style: 'danger', callback: (id) => {...} },
    { label: 'Cancel', style: 'secondary', callback: () => {} },
  ]
});

// Logger (toast + console output)
const logger = inject(ToastLogger);
logger.error('Request failed', { data: { status: 500 } });
